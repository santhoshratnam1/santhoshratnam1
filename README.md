I design **playable spaces and the tunable systems behind them** in Unreal Engine 5 — greybox to playable, plus the data-driven Blueprint systems that let a whole team tune a game without ever touching the graph.

Most of my work lives at the intersection of **level design and technical design**: sightline-guided levels, encounter pacing, and self-documenting, designer-friendly gameplay systems. Three years hands-on in Unreal Engine Blueprints.

My [**portfolio**](https://santhoshratnam1.github.io/SanthoshPortfolio/) collects shipped and greybox work; I'm currently building a rolling set of focused UE5 pieces across both design tracks. Open to **Level / Technical / Game Designer** roles.

**What I work on**

- **Level design**: blockout to playable, sightline-guided flow, verticality, encounter pacing, spatial storytelling
- **Technical design**: Blueprint gameplay systems, data-driven content (DataTables / DataAssets), StateTree AI, replication
- **Designer tooling**: Editor Utility Widgets and PCG-based tools so other designers can build and tune without code
- **Combat & encounters**: cover/sightline authoring, wave and reinforcement pacing, tension curves
- **The bar I hold**: self-documenting, clamped, and data-driven — a designer should retune my systems from data alone
